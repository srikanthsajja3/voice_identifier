# Urban Noise Identifier & XAI - Technical Documentation

## 1. Project Overview
This project is an advanced Audio Classification system designed to identify urban noise sources (e.g., sirens, drilling, dog barks) using a 2D Convolutional Neural Network (CNN). Beyond standard classification, it integrates **Explainable AI (XAI)** to show *why* the model made a specific prediction by highlighting key regions in the frequency spectrum.

---

## 2. The AI Model (CNN Architecture)
The model is a deep learning architecture optimized for image-like audio data (Mel-Spectrograms).

- **Input Layer:** Accepts 128x173 single-channel images (Mel-Spectrograms).
- **Convolutional Blocks:**
    - **Block 1:** 32 filters, ReLU activation, Batch Normalization, MaxPooling.
    - **Block 2:** 64 filters, ReLU activation, Batch Normalization, MaxPooling.
    - **Block 3 (XAI Target):** 128 filters, ReLU activation. This layer (`last_conv`) is the target for the Grad-CAM algorithm.
- **Output Layer:** A Global Average Pooling layer followed by a Dense layer with Softmax activation, outputting probabilities for 10 classes.

---

## 3. Audio Feature Engineering
Before the AI "hears" the sound, the raw waveform is converted into several visual and mathematical formats:

1. **Mel-Spectrograms:** The primary input for the AI. It maps frequency on the Y-axis and time on the X-axis, using a logarithmic scale to mimic human hearing.
2. **MFCCs (Mel-Frequency Cepstral Coefficients):** These represent the "texture" of the sound. They are crucial for distinguishing between sounds with similar frequencies but different characters (like a car horn vs. a whistle).
3. **Time-Domain Waveform:** Shows the raw amplitude of the signal over time, useful for identifying sharp transients (gunshots) vs. steady noise (air conditioners).

---

## 4. Explainable AI (Grad-CAM)
**Grad-CAM (Gradient-weighted Class Activation Mapping)** is the core XAI feature.
- **Process:** It calculates the gradients of the predicted class with respect to the feature maps of the `last_conv` layer.
- **Heatmap:** It generates a heatmap where "red" regions indicate the exact frequencies and timestamps that most influenced the AI's decision.
- **Utility:** This allows engineers to verify that the model is looking at the "siren" sound rather than background wind noise.

---

## 5. Performance Metrics & Visualizations
The system provides several diagnostics to ensure accuracy:

- **Confusion Matrix:** A 10x10 grid showing where the model is accurate and where it gets "confused" (e.g., mistaking a jackhammer for a drill).
- **XAI Heatmap Matrix:** A unique 2x5 grid that displays the AI's "attention" for a sample from every single class simultaneously.
- **Probability Distribution:** A real-time bar chart showing the confidence score for all possible outcomes.
- **Training Metrics:** Accuracy and Loss curves that track the model's learning progress over time.

## 6. Web Dashboard (Flask)
The dashboard provides a user-friendly interface for real-time interaction:
- **Real-time Recording:** Records 4 seconds of audio directly from the browser.
- **Instant Analysis:** Automatically processes the audio and updates all charts and heatmaps.
- **Smart City Trends:** A heuristic-based module that estimates future noise levels for urban planning.
